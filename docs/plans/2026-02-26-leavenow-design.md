# LeaveNow 기획서

## 개요

경기도 등 장거리 통근자를 위한 버스/지하철 출발 시간 알림 앱.
집→회사, 회사→집 두 시나리오에서 다음 출발 교통편 2~3개를 메뉴바/위젯으로 빠르게 확인할 수 있다.

---

## 문제 정의

- 경기도 등 수도권 외곽 거주자는 버스 배차 간격이 15~30분 이상
- 놓치면 오래 기다려야 하므로 미리 파악하는 것이 중요
- 기존 앱(카카오맵, 네이버지도)은 매번 직접 검색해야 해서 불편

---

## 타겟 사용자

- 경기도·인천 등 수도권 외곽에서 서울로 출퇴근하는 직장인
- Mac 사용자 (1차), iPhone/Android 사용자 (2차)

## 핵심 기능

### 1. 시나리오 자동 감지

출근/퇴근 시나리오를 아래 우선순위로 자동 판단한다.

```
1순위: GPS 위치
  - 집 반경 500m 이내 → 출근 모드 (집 → 회사)
  - 회사 반경 500m 이내 → 퇴근 모드 (회사 → 집)

2순위: 시간 기준 (GPS 실패/범위 밖일 때)
  - 오전 ~ 오후 3시 → 출근 모드
  - 오후 3시 이후 → 퇴근 모드

3순위: 수동 오버라이드
  - 메뉴바 클릭 → 수동으로 모드 전환 가능
```

### 2. 교통편 정보 표시

메뉴바에 다음 출발 교통편 **2~3개**를 표시한다.

**메뉴바 (평상시):**
```
🚌 9401  8분  |  🚇 2호선 15분  |  🚌 9401 29분
```

**클릭 시 드롭다운:**
```
[출근 모드]  ↔ 퇴근으로 전환
─────────────────────────
🚌 9401      8분 후  (오전 8:42)
🚇 2호선    15분 후  (오전 8:49)
🚌 9401     29분 후  (오전 9:03)
─────────────────────────
[설정]  [새로고침]
```

### 3. 설정

- 집 주소 입력 (네이버 지도 자동완성)
- 회사 주소 입력 (네이버 지도 자동완성)
- 시간 기준 기본값 조정 (오후 3시 → 사용자 지정 가능)

## 기술 설계

### 아키텍처

```
leavenow/
├── lib/
│   ├── core/
│   │   ├── services/
│   │   │   ├── naver_transit_service.dart   # 네이버 API 호출 + 캐싱
│   │   │   ├── location_service.dart        # GPS 위치
│   │   │   └── scenario_service.dart        # 출근/퇴근 시나리오 판단
│   │   └── models/
│   │       ├── transit_route.dart           # 경로 데이터
│   │       └── departure.dart              # 출발 정보 (노선, 남은 시간)
│   └── platforms/
│       └── macos/
│           └── menu_bar_app.dart           # macOS 메뉴바 UI
└── docs/
    └── plans/
        └── 2026-02-26-leavenow-design.md   # 본 문서
```

### API

- **네이버 지도 API** (Directions - 대중교통 경로 탐색)
- Client ID: 환경변수로 관리 (`.env`, `.gitignore` 처리 필수)

### API 갱신 전략 (비용 최적화)

| 이벤트 | 동작 |
|--------|------|
| 앱 실행 | 경로 탐색 1회 호출 |
| 시나리오 전환 | 경로 탐색 1회 재호출 |
| 버스 출발 2분 전 | 해당 편만 갱신 |
| 수동 새로고침 | 즉시 1회 호출 |

→ **하루 예상 호출: 10~20건** (무료 쿼터 1,000건 대비 여유)
